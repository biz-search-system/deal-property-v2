## 1. 機能概要

### 1.1 目的

複数の組織（会社・チーム）を管理し、ユーザーの所属組織に基づいたアクセス制御を実現するシステムです。Better Auth の Organization プラグインを基盤として、組織・メンバー・チーム・招待の管理機能を提供します。

### 1.2 主要機能

| 機能 | 説明 |
|------|------|
| 組織一覧 | ユーザーが所属する全組織を表示 |
| 組織作成 | 新しい組織を作成する |
| 組織管理 | 組織の設定・削除を行う |
| メンバー管理 | メンバーの一覧表示・役割変更・削除 |
| 招待管理 | 新規メンバーの招待・招待状の管理 |
| チーム管理 | 組織内のチーム作成・編集・メンバー管理 |

### 1.3 画面構成

```
/organization
├── /                              # 組織一覧
├── /new                           # 新規組織作成
└── /[organizationId]              # 組織管理（タブUI）
    ├── Members                    # メンバー管理タブ
    ├── Invitations                # 招待一覧タブ
    ├── Invite                     # 新規招待タブ
    └── Teams                      # チーム管理タブ
```

## 2. データモデル

### 2.1 主要テーブル

| テーブル名 | 説明 |
|-----------|------|
| organizations | 組織の基本情報 |
| members | 組織とユーザーの紐付け（メンバーシップ） |
| invitations | 招待情報 |
| teams | チーム情報 |
| team_members | チームとユーザーの紐付け |

### 2.2 組織（organizations）

| 項目名 | 項目ID | 型 | 必須 | 説明 |
|--------|--------|-----|------|------|
| ID | id | string | ○ | 一意識別子 |
| 組織名 | name | string | ○ | 組織の表示名 |
| スラッグ | slug | string | ○ | URL用識別子（ユニーク） |
| ロゴ | logo | string | - | 組織ロゴのURL |
| メタデータ | metadata | text | - | 追加情報（JSON） |
| 並び順 | sortOrder | integer | - | 表示順序（デフォルト: 0） |
| 作成日時 | createdAt | timestamp | - | 作成日時 |

### 2.3 メンバー（members）

| 項目名 | 項目ID | 型 | 必須 | 説明 |
|--------|--------|-----|------|------|
| ID | id | string | ○ | 一意識別子 |
| 組織ID | organizationId | string | ○ | 所属組織 |
| ユーザーID | userId | string | ○ | ユーザー参照 |
| 役割 | role | enum | ○ | owner/admin/member |
| 作成日時 | createdAt | timestamp | - | 加入日時 |

### 2.4 招待（invitations）

| 項目名 | 項目ID | 型 | 必須 | 説明 |
|--------|--------|-----|------|------|
| ID | id | string | ○ | 一意識別子 |
| 組織ID | organizationId | string | ○ | 招待先組織 |
| メール | email | string | ○ | 招待先メールアドレス |
| 役割 | role | enum | ○ | 招待時の役割 |
| チームID | teamId | string | - | 招待先チーム（任意） |
| ステータス | status | enum | ○ | pending/accepted/rejected |
| 招待者ID | inviterId | string | ○ | 招待を送った人 |
| 有効期限 | expiresAt | timestamp | ○ | 招待の有効期限 |
| 作成日時 | createdAt | timestamp | - | 招待日時 |

### 2.5 チーム（teams）

| 項目名 | 項目ID | 型 | 必須 | 説明 |
|--------|--------|-----|------|------|
| ID | id | string | ○ | 一意識別子 |
| 組織ID | organizationId | string | ○ | 所属組織 |
| チーム名 | name | string | ○ | チームの表示名 |
| 作成日時 | createdAt | timestamp | - | 作成日時 |
| 更新日時 | updatedAt | timestamp | - | 更新日時 |

### 2.6 チームメンバー（team_members）

| 項目名 | 項目ID | 型 | 必須 | 説明 |
|--------|--------|-----|------|------|
| ID | id | string | ○ | 一意識別子 |
| チームID | teamId | string | ○ | 所属チーム |
| ユーザーID | userId | string | ○ | ユーザー参照 |
| 作成日時 | createdAt | timestamp | - | 加入日時 |

## 3. 役割（Roles）

### 3.1 役割の定義

| 値 | 表示名 | 説明 |
|----|--------|------|
| owner | オーナー | 組織の所有者。全権限を持つ |
| admin | 管理者 | メンバー・チーム管理権限を持つ |
| member | メンバー | 閲覧権限のみ |

### 3.2 権限マトリクス

| 操作 | owner | admin | member |
|------|-------|-------|--------|
| 組織情報の閲覧 | ○ | ○ | ○ |
| 組織情報の編集 | ○ | - | - |
| 組織の削除 | ○ | - | - |
| メンバーの閲覧 | ○ | ○ | ○ |
| メンバーの招待 | ○ | ○ | - |
| メンバーの役割変更 | ○ | ○ | - |
| メンバーの削除 | ○ | ○ | - |
| チームの作成 | ○ | ○ | - |
| チームの編集 | ○ | ○ | - |
| チームの削除 | ○ | ○ | - |
| チームメンバーの管理 | ○ | ○ | - |
| 組織からの退出 | ○* | ○ | ○ |

**注**: オーナーは他のオーナーがいない限り退出できません

## 4. リレーション

### 4.1 ERダイアグラム

```
users (1) ─────────────────── (N) members ── (1) organizations
  │                                              │
  │                                              │
  ├── (N) team_members ── (1) teams ────────────┤
  │                                              │
  └── (N) invitations ────────────────────── (1)┘
                              │
                              └── (N) teams (optional)
```

### 4.2 リレーション定義

- `organizations` ↔ `members` : 1対多（組織には複数メンバー）
- `organizations` ↔ `teams` : 1対多（組織には複数チーム）
- `organizations` ↔ `invitations` : 1対多（組織には複数招待）
- `users` ↔ `members` : 1対多（ユーザーは複数組織に所属可能）
- `users` ↔ `team_members` : 1対多（ユーザーは複数チームに所属可能）
- `teams` ↔ `team_members` : 1対多（チームには複数メンバー）
- `users` ↔ `invitations` : 1対多（招待者として）

## 5. 招待ステータス

| 値 | 表示名 | 説明 |
|----|--------|------|
| pending | 保留中 | 招待送信済み、未応答 |
| accepted | 承諾済み | 招待を受け入れ済み |
| rejected | 拒否済み | 招待を拒否済み |

## 6. アクティブ組織

### 6.1 概要

ユーザーは複数の組織に所属可能であり、現在操作対象となる「アクティブ組織」を切り替えることができます。

### 6.2 仕様

- ユーザーごとに1つのアクティブ組織を保持
- セッションに紐づいて管理
- 案件管理などのデータはアクティブ組織でフィルタリング
- オーナーは「全社」を選択して全組織のデータを表示可能
