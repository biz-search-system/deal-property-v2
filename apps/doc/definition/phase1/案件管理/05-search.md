## 1. 画面概要

### 1.1 画面の目的

全案件を検索・フィルタリングする画面です。あいまい検索（Fuse.js）を使用して、物件名・オーナー名・担当者名などで横断的に検索できます。

### 1.2 想定利用者

- 営業担当者
- 事務担当者
- 経営者

### 1.3 画面のURL

- 検索一覧: `/properties/search`
- 詳細: `/properties/search/[id]`

## 3. 検索機能

### 3.1 キーワード検索

Fuse.jsを使用したあいまい検索。

| 検索対象フィールド | 説明 |
|-------------------|------|
| propertyName | 物件名 |
| ownerName | オーナー名 |
| roomNumber | 号室 |
| buyerCompany | 買取業者 |
| notes | 備考 |
| staff.name | 担当者名 |
| organization.name | 組織名 |

### 3.2 Fuse.js設定

```typescript
const fuseOptions = {
  keys: [
    'propertyName',
    'ownerName',
    'roomNumber',
    'buyerCompany',
    'notes',
    'staff.name',
    'organization.name'
  ],
  threshold: 0.3, // あいまい度（0が完全一致、1が全て一致）
  ignoreLocation: true,
  includeScore: true
};
```

### 3.3 検索の動作

- 入力と同時にリアルタイムで検索（デバウンス: 300ms）
- 検索結果が0件の場合、「該当する案件がありません」と表示
- 検索をクリアすると全件表示に戻る

## 4. フィルター機能

### 4.1 フィルター項目

| フィルター | 選択肢 | 複数選択 |
|-----------|--------|----------|
| 組織 | 全組織 / レイジット / エスク / TOUSEI | × |
| 進捗 | 全て / 7段階のステータス | ○ |
| 書類 | 全て / 3段階のステータス | ○ |
| 契約形態 | 全て / 9種類の契約形態 | ○ |

### 4.2 フィルターの動作

- フィルターは検索と組み合わせて使用可能
- 複数のフィルターはAND条件で適用
- フィルターをリセットするボタンを提供

### 4.3 フィルターコンポーネント

DataTableToolbarコンポーネントを使用。

```
┌─────────────────────────────────────────────────────────────┐
│ [組織▼] [進捗▼] [書類▼] [契約形態▼]  [フィルターをリセット]   │
└─────────────────────────────────────────────────────────────┘
```

## 7. データフィルタリング

### 7.1 表示条件

- 進捗ステータスが「BC確定前」（bc_before_confirmed）**以外**の案件のみ表示
- ユーザーの所属組織に基づくフィルタリング

### 7.2 権限によるフィルタリング

| ロール | 表示範囲 |
|--------|----------|
| owner（経営者） | Active Organization = null: 全組織 / 特定: その組織のみ |
| admin（管理者） | 所属組織の案件のみ |
| member（メンバー） | 所属組織の案件のみ |

## 8. データフロー

### 8.1 初期データ取得

```
[ページロード]
    ↓
[Server Component]
    ↓
[getAllPropertiesBySettlementDate()]
    - organizationIds でフィルタリング
    - progressStatus !== 'bc_before_confirmed'
    - settlementDate ASC でソート
    ↓
[SearchProperties（Client Component）]
    ↓
[Fuse.jsインスタンス作成]
    ↓
[レンダリング]
```

### 8.2 検索実行

```
[検索キーワード入力]
    ↓
[デバウンス（300ms）]
    ↓
[Fuse.js検索]
    ↓
[検索結果でテーブル更新]
```

### 8.3 フィルター適用

```
[フィルター選択]
    ↓
[Zustand状態更新]
    ↓
[フィルタリング処理]
    ↓
[Fuse.js検索と組み合わせ]
    ↓
[テーブル更新]
```

## 11. パフォーマンス考慮事項

### 11.1 クライアントサイド検索

- Fuse.jsはクライアントサイドで動作
- 初期データは全件取得してメモリに保持
- 大量データの場合、初期ロードに時間がかかる可能性あり
