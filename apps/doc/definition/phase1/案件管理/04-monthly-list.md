## 1. 画面概要

### 1.1 画面の目的

指定した月の決済予定案件および決済完了案件を一覧表示する画面です。月ごとの利益合計やBC手付合計を集計表示し、決済スケジュールを管理できます。

### 1.2 想定利用者

- 営業担当者
- 事務担当者
- 経営者

### 1.3 画面のURL

- 一覧: `/properties/monthly/[year]/[month]`
- 詳細: `/properties/monthly/[year]/[month]/[id]`

例: `/properties/monthly/2026/01`（2026年1月の案件一覧）

## 4. データフィルタリング

### 4.1 表示条件

以下の条件を満たす案件を表示:

1. 決済日（settlementDate）が選択した月の範囲内
2. 進捗ステータスが「BC確定前」（bc_before_confirmed）**以外**

### 4.2 月の範囲判定

日本時間（JST）を基準に月の範囲を判定。

```
例: 2026年1月の場合
開始: 2026-01-01 00:00:00 JST
終了: 2026-01-31 23:59:59 JST
```

### 4.3 権限によるフィルタリング

| ロール | 表示範囲 |
|--------|----------|
| owner（経営者） | Active Organization = null: 全組織 / 特定: その組織のみ |
| admin（管理者） | 所属組織の案件のみ |
| member（メンバー） | 所属組織の案件のみ |

### 4.4 ソート

決済日（settlementDate）の昇順でソート。

## 7. データフロー

### 7.1 初期データ取得

```
[ページロード]
    ↓
[Server Component]
    ↓
[getMonthlyPropertiesByOrganizations(year, month)]
    - organizationIds でフィルタリング
    - progressStatus !== 'bc_before_confirmed'
    - settlementDate が当月内
    - settlementDate ASC でソート
    ↓
[MonthlyPropertiesTable]
    ↓
[集計計算]
    - 利益合計
    - BC手付合計
    - 件数
    ↓
[レンダリング]
```

### 7.2 月切り替え

```
[前月/次月ボタンクリック]
    ↓
[URLパラメータ更新]
    /properties/monthly/[year]/[month]
    ↓
[ページリロード]
    ↓
[新しい月のデータ取得]
```

### 7.3 インライン編集（決済日変更）

```
[決済日セルクリック]
    ↓
[DatePicker表示]
    ↓
[日付選択]
    ↓
[Server Action: updatePropertySettlementDate]
    ↓
[revalidatePath('/properties/monthly/[year]/[month]')]
    ↓
[UI更新]
    ↓
[別の月に移動した場合、一覧から消える]
```
