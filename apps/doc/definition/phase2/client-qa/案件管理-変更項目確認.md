## 確認のお願い

以下の追加項目について、**表示名**・**選択肢**・**デフォルト値** が正しいかご確認ください。

将来的なリスト表示や一覧画面での視認性を考慮し、**選択肢の文字数**を3〜5文字に短縮することを提案します。一部の項目で「現在案」と「修正案（短縮形）」を併記していますので、どちらが適切かご確認ください。

### ステータスの統一化について

特殊な事例を除き、進捗管理のステータスを以下のように統一することを提案します。

**従来のステータス（4つ）**:
未依頼 → 依頼中 → 取得済 → 不要

**新しいステータス（5つ）**:
未依頼 → 確認中 → 対応中 → 完了 → 不要

| ステータス | 説明 |
|------------|------|
| 未依頼 | まだ対応を開始していない |
| 確認中 | 社内で調整・確認作業中（期日調整、方法確認など） |
| 対応中 | 社外（銀行、司法書士、管理会社等）に依頼済み、対応待ち |
| 完了 | 対応完了 |
| 不要 | この案件では対応不要 |

**変更のメリット**:

1. **進捗の可視化向上**: 「依頼中」を「確認中」と「対応中」に分けることで、社内作業なのか社外待ちなのかが一目でわかる
2. **ボトルネックの特定**: 社内確認で止まっているのか、社外対応待ちなのかを区別でき、対応が遅れている箇所を特定しやすい
3. **統一的な運用**: 異なる項目間で同じステータス体系を使うことで、操作の学習コストが下がり、一覧画面での視認性も向上
4. **汎用性**: 「取得済」「手続き完了」などを「完了」に統一することで、どの項目でも同じ意味で使える
