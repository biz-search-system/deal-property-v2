# 案件管理機能 変更仕様書（確定版）

## バージョン情報

- **作成日**: 2026-01-17
- **確定日**: 2026-01-24
- **バージョン**: 1.1
- **対象**: 案件編集画面（決済進捗タブ）

---

## 1. 変更概要

案件管理の編集画面（決済進捗タブ）において、以下のカテゴリの項目を追加・変更します。

| カテゴリ     | 変更内容               |
| ------------ | ---------------------- |
| 銀行関連     | 項目追加、画面位置移動 |
| 司法書士関連 | 項目追加               |
| 手出し関係   | 新規カテゴリ追加       |
| 賃管関係     | 項目追加               |

### 1.1 ステータスの統一化

進捗管理のステータスを以下の5段階に統一します。

| ステータス | 説明                                                   |
| ---------- | ------------------------------------------------------ |
| 未依頼     | まだ対応を開始していない                               |
| 確認中     | 社内で調整・確認作業中（期日調整、方法確認など）       |
| 対応中     | 社外（銀行、司法書士、管理会社等）に依頼済み、対応待ち |
| 完了       | 対応完了                                               |
| 不要       | この案件では対応不要                                   |

## 3. 司法書士関連

### 3.1 変更内容

決済進捗タブの司法書士関連セクションに以下の項目を追加します。

### 3.2 権利証（propertyTitle）

売主が権利証（登記識別情報）を所持しているかを確認する項目です。
権利証を紛失している場合は、別途司法書士との立会による本人確認が必要となります。

| 項目       | 値                 |
| ---------- | ------------------ |
| 項目ID     | propertyTitle      |
| 項目名     | 権利証             |
| 型         | boolean (nullable) |
| 必須       | -                  |
| デフォルト | null               |

#### 値

| 値    | 表示名 | 説明                       |
| ----- | ------ | -------------------------- |
| true  | 有     | 売主が権利証を所持している |
| false | 無     | 売主が権利証を紛失している |
| null  | 未設定 | 未確認                     |

### 3.3 住所変更（addressChange）

売主が売買契約時点で、購入時の登記住所から住所を変更しているかを確認する項目です。
住所変更がある場合は、別途住所変更登記手続きが必要となります。

| 項目       | 値                 |
| ---------- | ------------------ |
| 項目ID     | addressChange      |
| 項目名     | 住所変更           |
| 型         | boolean (nullable) |
| 必須       | -                  |
| デフォルト | null               |

#### 値

| 値    | 表示名 | 説明                                           |
| ----- | ------ | ---------------------------------------------- |
| true  | 有     | 売主が購入時の登記住所から住所を変更している   |
| false | 無     | 売主が購入時の登記住所から住所を変更していない |
| null  | 未設定 | 未確認                                         |

### 3.4 氏名変更（nameChange）

売主が売買契約時点で、購入時の登記氏名から氏名（苗字等）を変更しているかを確認する項目です。
氏名変更がある場合は、別途氏名変更登記手続きが必要となります。

| 項目       | 値                 |
| ---------- | ------------------ |
| 項目ID     | nameChange         |
| 項目名     | 氏名変更           |
| 型         | boolean (nullable) |

| デフォルト | null               |

#### 値

| 値    | 表示名 | 説明                                           |
| ----- | ------ | ---------------------------------------------- |
| true  | 有     | 売主が購入時の登記氏名から氏名を変更している   |
| false | 無     | 売主が購入時の登記氏名から氏名を変更していない |
| null  | 未設定 | 未確認                                         |

### 3.5 本人確認方法（identityVerificationMethod）

## 4. 手出し関係（新規カテゴリ）

### 4.1 変更内容

決済進捗タブに「手出し関係」セクションを新規追加します。

### 4.2 手出し状況（sellerFundingStatus）

売主がローン残債と売却金額の差額（手出し金）を用意するための進捗を管理します。

**業務背景**:

- ローン残債 > 売却金額 の場合、売主は差額分の手出し金を用意する必要がある
- 自己資金で完結できれば「不要」
- 借入が必要な場合は親族・友人・フリーローン等から調達
- 本審査完了で借入可能額が確定し、借入完了・手出し金用意で「用意完了」となる

| 項目       | 値                  |
| ---------- | ------------------- |
| 項目ID     | sellerFundingStatus |
| 項目名     | 手出し状況          |
| 型         | enum                |
| 必須       | -                   |
| デフォルト | not_required        |

#### 選択肢

| 値                 | 表示名   | 説明                                               |
| ------------------ | -------- | -------------------------------------------------- |
| not_required       | 不要     | 売却金額でローン残債をカバーできるため、手出し不要 |
| preliminary_review | 仮審査中 | 売主がローンを申請し、金融機関による仮審査を実施中 |
| final_review       | 本審査中 | 仮審査通過後、金融機関による本審査を実施中         |
| review_completed   | 審査完了 | ローン本審査が完了し、借入可能額が確定             |
| funds_ready        | 用意完了 | 売主が手出し金の用意を完了し、決済に臨める状態     |

## 8. 実装時の注意事項

### 8.1 既存項目との整合性

- 既存の `identityVerification` は「本人確認書類」として残す
- 新規の `identityVerificationStatus` は「本人確認」として追加
- 既存の `guaranteeTransferStatus` は「保証会社承継ステータス」として残す
- 新規の `guaranteeCompanySuccession` は「保証会社承継」として追加

### 8.2 書類進捗タブからの移動

- ローン計算書（loan_calculation）を書類進捗タブから削除
- 決済進捗タブの銀行関係セクションに追加

### 8.3 UI実装

- 新規項目はSelect コンポーネントで実装
- 有/無の選択は Badge Toggle パターンも検討可能
- 既存のインライン編集パターンに準拠
- 表示名は短縮形（3〜5文字）を採用
