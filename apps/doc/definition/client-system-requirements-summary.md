## 2. 共通要件

### 2.1 データ保護

- 誤削除防止（論理削除、確認ダイアログ）
- 変更履歴の保持

### 2.2 日付管理

- 各工程の開始日、完了日を記録
- タイムスタンプの自動記録

### 2.3 進捗管理

- ステータスの一元管理
- ワンクリックでの状態遷移
- 一覧での進捗可視化

### 2.4 アラート機能

- 期限切れ/遅延のアラート
- 未対応タスクの通知

## 3. ユーザー種別

| ユーザー | 主な操作 |
|----------|----------|
| 営業 | AB契約、管理解約申込、ローン計算書取得依頼 |
| 事務 | マイソク作成、調査、BC契約書作成、精算書作成、司法書士依頼 |
| 管理者 | 売上管理、全体進捗確認 |
| 司法書士（外部） | 進捗入力、資料閲覧（共有機能が実装された場合） |

## 4. データエンティティ（想定）

### 4.1 主要エンティティ

```
Property（物件）
├── PropertyInquiry（業者配布・金額回答）
├── ABContract（AB契約）
├── Investigation（物件調査）
│   └── InvestigationDocument（調査書類）
├── BCContract（BC契約）
├── Settlement（精算書）
├── LawyerRequest（司法書士依頼）
└── RentalManagementCancellation（賃貸管理解約）

Broker（買取業者）
├── BrokerResponse（金額回答）
└── BrokerAnalytics（業者分析）

User（ユーザー）
├── SalesStaff（営業）
└── AdminStaff（事務）

SalesRecord（売上記録）
```

## 5. 開発優先順位の提案

### Phase 1: 基盤構築

1. 物件マスタ管理
2. AB契約管理
3. BC契約管理
4. 基本的な進捗ステータス管理

### Phase 2: 調査・精算

1. 物件調査管理
2. 精算書管理
3. 日付記録・履歴機能

### Phase 3: 外部連携・分析

1. 司法書士依頼管理
2. 賃貸管理解約管理
3. アラート機能

### Phase 4: 業者配布・売上

1. マイソク配布システム
2. 金額回答フォーム
3. 売上管理・分析
