# ビジネス概要の整理

## 1. ビジネスモデルの本質

### 収益構造
- **利益源①**: 仲介手数料（AB契約時）- 約39.6万円
- **利益源②**: 売買差益（BC契約時）- 約200万円
- **合計利益**: 約239.6万円/案件

### 取引フローの核心
1. テレアポで物件所有者（A）にアプローチ
2. グループ会社（B）を通じて買取
3. 買取業者（C）に転売
4. BC決済完了で業務終了（CD取引は関与なし）

## 2. 現状の課題（システム化の背景）

### データ管理の問題
- スプレッドシート管理による誤削除リスク
- 複数箇所への情報分散（スプレッドシート、Evernote等）
- 手入力による入力ミスの発生

### 業務効率の問題
- 進捗状況の把握困難
- 担当者間での情報共有の困難
- データの正確性低下

## 3. MVPで解決すべき最重要課題

1. **データの一元管理**
   - 案件情報の統合管理
   - 誤削除の防止

2. **進捗の可視化**
   - 業務ステータス（7段階）の明確化
   - 書類ステータス（3段階）の追跡

3. **権限管理**
   - 組織別のアクセス制御
   - 金額情報の編集制限（事務組織）

## 4. 売上計上ルールの重要性

### 基本ルール
- 当月契約 → 翌月末までにC確定 = 当月売上
- 例: 8月15日契約
  - 9月27日にC確定 → 8月売上
  - 10月3日にC確定 → 9月売上

このルールはシステムの月次集計機能に直結する重要な仕様。
